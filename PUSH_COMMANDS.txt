# Manual GitHub Push Commands

## Prerequisites
1. Create repository on GitHub: https://github.com/new
   - Name: tender-management-system
   - Visibility: Private (recommended)
   - DO NOT initialize with README

2. Install Git if not already installed:
   - Download: https://git-scm.com/download/win

## Commands to Run

Replace `YOUR_USERNAME` with your actual GitHub username:

```bash
cd c:\Users\Oumer\Desktop\auction

git init

git add .

git commit -m "Initial commit: Tender Management System"

git remote add origin https://github.com/YOUR_USERNAME/tender-management-system.git

git branch -M main

git push -u origin main
```

## Authentication

When prompted for credentials:
- **Username**: Your GitHub username
- **Password**: Personal Access Token (NOT your GitHub password)

### Generate Personal Access Token:
1. Go to: https://github.com/settings/tokens
2. Click "Generate new token" → "Generate new token (classic)"
3. Name: `Tender Management Deploy`
4. Expiration: 90 days (or custom)
5. Select scopes: ✅ `repo` (full control of private repositories)
6. Click "Generate token"
7. **COPY THE TOKEN** (you won't see it again!)
8. Use this token as your password when pushing

## Verify Success

After pushing, visit:
https://github.com/YOUR_USERNAME/tender-management-system

You should see all your files!
